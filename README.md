# Code and materials for “From network learning to improved permutation testing: addressing statistical challenges in microbial data analysis”

This repository contains code, generated outputs, and chapter figures for the PhD thesis *From network learning to improved permutation testing: addressing statistical challenges in microbial data analysis*. The repository is organized by thesis chapters: each `chXX_*` folder collects the material that belongs to the corresponding chapter.

Some chapter folders mainly store figures used in the thesis, while others also contain reproducible R Markdown workflows, simulation code, intermediate data objects, and rendered reports.

## Reproducibility

The project uses R and R Markdown. Package dependencies are managed with `renv`.

To restore the R environment, open the project in RStudio or start R in the repository root and run:

```r
renv::restore()
```

Individual R Markdown files can be rendered from their chapter folders, for example:

```r
rmarkdown::render("ch02_microbiome_data/CLR_mini_example.Rmd")
rmarkdown::render("ch07_application/04_beta_diversity.Rmd")
```

The helper script `projthis_code.R` contains project-level rendering utilities for workflows that follow the `projthis` structure.

## Data Notes

All data required to reproduce the reported results are available in this repository. However, some large files containing simulated data are not included. In those cases, the corresponding folder should be rerun from its first script to regenerate the missing data before executing the later analysis scripts.

Depending on the analysis, regenerating simulated data and rerunning downstream scripts may require substantial computation time. Exact numerical values may differ slightly across systems because of floating-point arithmetic, package versions, or parallel execution.

## Working Conventions

- Chapter folders are the primary organizational unit.
- Figures used in the thesis should live in the relevant chapter's `figures/` folder.
- Reproducible analyses should be kept close to the chapter they support.
- Rendered `.html`, `.md`, `.pdf`, tables, and `.rds` files are retained when they document the outputs used in the thesis.
