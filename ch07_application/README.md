# Chapter 7: Integrated analysis of the PASTURE cohort

This folder contains the microbiome application analysis for Chapter 7 of the thesis. The analysis uses gut microbiota data from the PASTURE birth cohort, focusing on samples collected at 2 months of age from infants in Germany, Switzerland, and Austria.

The workflow is written as a sequence of R Markdown files. Each file can be rendered separately, but later files may depend on data objects created by earlier files.

## Folder Structure

```text
.
├── README.md                     → Chapter overview (this file)
├── 00_build_dataset.Rmd          → Build and prepare the analysis dataset
├── 01_overview.Rmd               → Descriptive cohort and microbiome overview
├── 02_filtering.Rmd              → Taxonomic aggregation and prevalence filtering
├── 03_alpha_diversity.Rmd        → Alpha diversity analysis
├── 04_beta_diversity.Rmd         → Beta diversity analysis
├── 05_regression.Rmd             → Regression models for compositional responses
├── 06_comp_equivalence.Rmd       → Compositional equivalence analysis
├── data/                         → Outputs generated by the R Markdown workflow
│   ├── 00_build_dataset/
│   ├── 01_overview/
│   ├── 02_filtering/
│   ├── 03_alpha_diversity/
│   ├── 04_beta_diversity/
│   ├── 05_regression/
│   ├── 06_comp_equivalence/
│   └── PASTURE_data/
└── figures/                      → Figures generated by the R Markdown workflow
    ├── 01_overview/
    ├── 02_filtering/
    ├── 03_alpha_diversity/
    ├── 04_beta_diversity/
    ├── 05_regression/
    └── 06_comp_equivalence/
```

Rendered `.html`, `.md`, and `.pdf` files are kept next to the corresponding R Markdown files when they document results used in the thesis.

## Workflow

The main analysis files are intended to be run in this order:

1. `00_build_dataset.Rmd`
2. `01_overview.Rmd`
3. `02_filtering.Rmd`
4. `03_alpha_diversity.Rmd`
5. `04_beta_diversity.Rmd`
6. `05_regression.Rmd`
7. `06_comp_equivalence.Rmd`

The first file builds the reduced analysis dataset. The following files use outputs stored in `data/` and write their own intermediate results, tables, and figures to matching chapter-specific subfolders.

## Rendering

From the repository root, source `projthis_code.R` and run the line for the file you want to render, for example:

```r
render_rmd("ch07_application/04_beta_diversity.Rmd")
```

The files can also be rendered directly with `rmarkdown::render()`, but using `render_rmd()` is recommended because it renders each file from its own folder and preserves the relative `data/` and `figures/` paths used by `projthis`.

## Notes

- All data required to reproduce the reported Chapter 7 results are available in this repository.
- Intermediate `.rds` objects, `.csv` summaries, and `.tex` tables are retained when they document outputs used in the thesis.
- Some analyses, especially resampling or permutation-based steps, may take substantial time to rerun.
